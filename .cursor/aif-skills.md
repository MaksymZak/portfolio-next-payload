# AI Factory (aif) — справочник скилов

Краткий справочник по slash-командам AI Factory в этом проекте: когда вызывать каждый скил и как они связаны в типичном пайплайне.

## Старт проекта и контекст

| Скил | Когда использовать |
|------|-------------------|
| `/aif` | Начало работы с проектом: анализ стека, установка скилов с [skills.sh](https://skills.sh), настройка MCP. «Настрой проект», «какие скилы нужны». |
| `/aif-architecture` | Выбор и фиксация архитектуры → `.ai-factory/ARCHITECTURE.md`. После `/aif` или при вопросе «какую архитектуру выбрать». |
| `/aif-roadmap` | Стратегический план: вехи, «что строить дальше» → `ROADMAP.md`. |
| `/aif-rules` | Добавить правило проекта («всегда делай X», «запомни конвенцию») → `RULES.md`. Подхватывается `/aif-implement`. |
| `/aif-reference` | Сохранить знания из URL/файлов в `.ai-factory/references/` для агента. |
| `/aif-skill-generator` | Создать новый Agent Skill (SKILL.md, references, scripts). |
| `/aif-distillation` | Превратить книгу, папку или URL в компактный скил (один или несколько). |

## Планирование и исследование

| Скил | Когда использовать |
|------|-------------------|
| `/aif-plan` | План реализации фичи. Режимы `fast` / `full`, опционально ветка/worktree. «Спланируй», «новая фича». |
| `/aif-explore` | Режим «подумать вместе»: идеи, проблемы, требования — до или во время изменений. |
| `/aif-improve` | Вторая итерация плана: пробелы, зависимости, лишние задачи. После `/aif-plan` или для плана из `/aif-fix`. |
| `/aif-grounded` | Ответы только с доказательствами, без догадок. «Будь уверен на 100%», высокие ставки. |

## Реализация

| Скил | Когда использовать |
|------|-------------------|
| `/aif-implement` | Выполнение плана по задачам, прогресс между сессиями. «Реализуй», «продолжи план». |
| `/aif-fix` | Баг или ошибка: быстрый фикс или план сначала. Без аргументов — `FIX_PLAN.md`. |
| `/aif-best-practices` | Справочник по качеству кода: именование, структура, ошибки, тесты. При написании и ревью. |
| `/aif-loop` | Итерации «сгенерировать → критика → доработка» до прохождения quality gates. |

## Проверка и качество

| Скил | Когда использовать |
|------|-------------------|
| `/aif-verify` | После `/aif-implement`: всё ли сделано, сборка, тесты. «Проверь работу». |
| `/aif-qa` | Тест-план и сценарии для фичи или ветки. «Что протестировать». |
| `/aif-review` | Ревью staged changes или PR: баги, безопасность, перформанс. |
| `/aif-rules-check` | Только проверка соблюдения `RULES.md` по diff/ref — без полного review. |
| `/aif-security-checklist` | Аудит по OWASP: auth, injection, XSS, секреты. Перед деплоем. |

## Инфраструктура и документация

| Скил | Когда использовать |
|------|-------------------|
| `/aif-ci` | CI/CD: GitHub Actions или GitLab CI (lint, тесты, security). |
| `/aif-dockerize` | Dockerfile, compose, `.dockerignore`, prod-аудит. |
| `/aif-build-automation` | Makefile, Taskfile, Justfile, Magefile — создать или дополнить. |
| `/aif-docs` | README + тематические docs. «Задокументируй проект». |

## Завершение и эволюция

| Скил | Когда использовать |
|------|-------------------|
| `/aif-commit` | Conventional Commits по staged changes. «Сделай коммит». |
| `/aif-archive` | Архив готовых планов, очистка roadmap. «Заархивируй планы». |
| `/aif-evolve` | Улучшение aif-скилов под проект по опыту и патчам. «Сделай AI умнее». |

## Типичный пайплайн

```
/aif → /aif-architecture → /aif-roadmap
         ↓
/aif-plan → /aif-improve (опционально)
         ↓
/aif-implement → /aif-verify → /aif-qa
         ↓
/aif-review + /aif-security-checklist → /aif-commit → /aif-archive
```

## Быстрые шпаргалки

- **Баг** → `/aif-fix`
- **Нужно подумать, не кодить** → `/aif-explore`
- **Нужна уверенность в фактах** → `/aif-grounded`
- **Правило «всегда так»** → `/aif-rules`
- **Документация из внешнего источника** → `/aif-reference` или `/aif-distillation`

## Скилы в этом репозитории

Файлы лежат в `.cursor/skills/aif*/SKILL.md`. Подробные инструкции для агента — внутри каждого `SKILL.md`.
