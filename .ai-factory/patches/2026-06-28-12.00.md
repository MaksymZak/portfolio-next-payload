# Console warnings on locale switch (React 19 script + scroll-behavior)

**Date:** 2026-06-28 12:00
**Files:** src/components/layout/theme-provider.tsx, src/app/(frontend)/[locale]/layout.tsx
**Severity:** low

## Problem

Switching locale (`router.replace` via LocaleSwitcher) logged two dev warnings:

1. `Encountered a script tag while rendering React component` — from `next-themes` re-injecting its inline theme script on client navigation.
2. `Detected scroll-behavior: smooth on the <html> element` — Next.js 16 expects `data-scroll-behavior="smooth"` when smooth scrolling is enabled.

## Root Cause

- `next-themes` renders `<script dangerouslySetInnerHTML>` inside a client `ThemeProvider`. React 19 warns when that runs during client-side renders (locale change remounts `[locale]/layout`).
- `<html>` used `motion-safe:scroll-smooth` without the opt-in attribute Next.js 16 uses to temporarily disable smooth scroll during route transitions.

## Solution

- Pass `scriptProps={{ type: 'application/json' }}` on the client only so React does not treat the tag as executable JS on navigations; SSR still emits the default blocking script for FOUC prevention.
- Add `data-scroll-behavior="smooth"` to `<html>` alongside `motion-safe:scroll-smooth`.

## Prevention

- When upgrading to React 19 / Next.js 16, audit `next-themes` (or migrate to a fork with `getThemeScript` + `noScript`).
- Any `scroll-smooth` on `<html>` needs `data-scroll-behavior="smooth"` per Next.js 16 upgrade guide.
