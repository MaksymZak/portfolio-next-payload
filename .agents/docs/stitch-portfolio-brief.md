# Stitch Portfolio Brief

Дата: 2026-06-01

## 1. Goal

Create a personal portfolio website for Maksym Zakaliuzhnyi.

The website should position Maksym as a reliable Middle Frontend / Next.js Developer with strong production landing page experience and practical CMS-driven website skills.

The portfolio is not a visual experiment and not a generic CV page. It should be a hiring tool for remote frontend roles in Ukrainian and European companies.

Primary impression:

> Reliable Middle Frontend / Next.js Developer for production landing systems and CMS-driven websites.

Secondary impression:

> Frontend developer with backend integration experience and growing fullstack skills.

## 2. Audience

Primary audience:

- recruiters;
- hiring managers;
- frontend/team leads;
- small teams looking for a production-focused frontend developer.

The page should help them understand within 10 seconds:

- Maksym is a commercial frontend developer, not a beginner;
- he has shipped and supported many production landing pages;
- he works with React, Next.js, TypeScript, Tailwind, Payload CMS;
- he can handle CMS-driven websites and production support;
- he is available for remote work.

## 3. Visual Direction

Style:

> Premium editorial technical portfolio.

The design should feel modern, polished, and frontend-oriented, but still serious and practical.

It should not look like a plain backend developer CV, but it also should not look like an overdesigned creative agency portfolio.

Design qualities:

- strong typography;
- clean grid;
- calm but premium visual rhythm;
- compact proof sections;
- polished cards for selected work;
- subtle hover states;
- subtle reveal/motion ideas;
- easy scanning;
- good mobile layout;
- good contrast and readable text;
- professional first impression.

Avoid:

- excessive animations;
- heavy 3D effects;
- decorative blobs/orbs;
- generic SaaS landing-page look;
- huge empty hero without useful proof;
- one-color purple/blue gradient theme;
- too many cards nested inside other cards;
- fake case study depth where details are not available yet.

## 4. Brand Feeling

Keywords:

- technical;
- production-ready;
- focused;
- clean;

- practical;
- confident;
- modern frontend.

The website should communicate:

## 10. SDD / Spec Kit Notes

This Stitch brief is not the final engineering spec.

Recommended process:

1. Use this document to get visual direction from Stitch.
2. Choose the best visual/layout ideas.
3. Convert the agreed direction into a `portfolio-mvp` specification.
4. Use Spec Kit or OpenSpec to clarify requirements, plan tasks, and split implementation.
5. Implement the project in small steps:
   - routes and i18n;
   - base layout and design system;
   - Payload collections;
   - homepage sections;
   - resume page;
   - PDF flow;
   - selected work content;
   - deployment.

The spec should describe what must exist and how success will be checked. It should not try to design every pixel.
